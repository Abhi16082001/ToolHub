# ToolHub Tool Inventory

Version: 1.0

---

# Purpose

This document is the single source of truth for all tools planned for ToolHub.

It defines:

- Categories
- Tool inventory
- Priority
- Difficulty
- SEO potential
- Implementation notes

All tools should comply with PROJECT.md, ARCHITECTURE.md, CONTENT.md and SEO.md.

---

# Project Rules

Every tool must satisfy all of the following:

✓ Runs entirely in the browser

✓ Requires no backend

✓ Requires no database

✓ Requires no paid API

✓ Can be deployed as a static Astro website

✓ Mobile friendly

✓ SEO friendly

✓ Accessible

✓ Solves one specific problem

---

# Status

Possible values:

- Planned
- In Progress
- Completed

# Priority

High

Implement as early as possible.

Medium

Implement after the core tools.

Low

Implement when the category is mature.

# Difficulty

Easy

Can usually be completed in under a few hours.

Medium

Requires moderate logic or algorithms.

Hard

Requires complex parsing, formatting or calculations.
