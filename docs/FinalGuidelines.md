
# Future Categories (Not Phase 1)

The following categories are intentionally postponed.

Do not implement unless explicitly requested.

- Image Tools
- PDF Tools
- File Tools
- CSS Visual Tools (if separated from Developer Tools)
- Color Analysis Tools
- Markdown Editors
- Date Utilities
- Browser Utilities

These categories should only be added after the five core categories are well established.

# Adding a New Tool

Before adding a new tool, verify:

✓ It solves one specific problem.

✓ It fits an existing category.

✓ It works completely in the browser.

✓ It does not require a backend.

✓ It does not require a paid API.

✓ It provides genuine value.

✓ It is not already covered by an existing tool.

✓ It follows the architecture defined in ARCHITECTURE.md.

# Long-Term Vision

The objective is not simply to build hundreds of tools.

The objective is to build one of the fastest, cleanest and most useful browser-based utility websites.

Every tool should:

- load quickly
- solve one problem well
- work without a backend
- follow the project architecture
- maintain a consistent user experience

Quality should always take priority over quantity.
